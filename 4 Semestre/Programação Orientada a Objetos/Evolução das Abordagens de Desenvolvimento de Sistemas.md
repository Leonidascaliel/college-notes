### Monolítico
* Década de 60
* Aplicações/Sistemas eram pequenos e relativamente simples
* Linguagens simples (Fortran, Cobol)
* Os desenvolvedores não tinham um método para desenvolver sistemas, utilizavam da sua criatividade pessoal
* A programação era considerada uma atividade artesanal e criativa
* Os códigos do tipo **Spaghetti** e de difícil manutenção

___
### Estruturado
* Década de 70
* **Al** **Constantine** e **Yourdon** criaram uma abordagem de desenvolvimento de sistemas que utilizavam a função como seu bloco de construção. Este método foi denominado **Análise e Desenho Estruturado**.
* Surgiu a **Programação modular** que facilitou a manutenção dos sistemas.
* Utilizavam de diagramas gráficos para representar os fluxos de dados (DFD = Diagrama de Fluxo de Dados) e para representar as lógicas de programação (Fluxogramas).
* Década de 80, **Peter** **Chen** (Criador do **diagrama de entidade e relacionamentos**) e **Ed** **Codd** (Desenhista do banco de dados relacional) Criando um método (Modelagem de dados) de desenvolvimento de sistemas baseados em dados.

___

### Orientado a Objetos
* Década de 90
* Visa complementar o método estruturado
* Fornece ao desenvolvedor uma **perspectiva** **ortogonal**, permitindo administrar a complexidade do domínio do problema e sua tecnologia de suporte
* Facilita a administração dos dados e o controle das funções (**código**)
* Permite a produção de software mais flexível e de fácil manutenção.

___

### Programação Orientada a Objetos
* A complexidade faz parte da essência do software
* Devemos utilizar de mecanismos que permitem administrar a complexidade do software
* A complexidade gera dificuldades
* Muitas dificuldades encontradas no desenvolvimento de um software podem surgir devido a forma empregada para construí-lo.

___

### Etapas para o Desenvolvimento de uma aplicação
1. Análise
2. Projeto
3. Implementação
4. Implantação
